## 浏览器的进程和线程

进程：负责多个模块的隔离，线程是依赖于进程的，每个线程处理特定的任务，进程通信成本（IPC通信成本高），这种设计保障了浏览器的稳定性、安全性和性能

比如浏览器的浏览器进程和渲染进程

渲染进程也就是页面的渲染过程：

GUI渲染线程解析DOM树和CSSOM树，然后合成渲染树，之后布局、渲染（重排重绘）

JS引擎线程负责处理js脚本，是单线程的，避免了多线程同时操作dom的问题



## tls协商

tls协商是建立再tcp连接之上的，显示通过加密算法生成一个随机数存放在客户端，然后客户端携带传递给服务端，服务端响应一个随机数和CA数字证书返回给客户端，客户端收掉ca数字证书之后破解私钥之后完成处理并且通知服务端，服务端通过客户端的私钥进行操作完成处理

## 作用域：全局作用域、函数作用域、块级作用域，使用来标识一个变量或者函数有效的代码块

闭包：简单来说是函数里面嵌套函数，实现了函数内部能够访问到外部作用域中的变量

this的指向一般分为：默认绑定、显示绑定和隐式绑定

默认绑定：比如全局作用域window中声明一个函数，然后在全局作用域中进行一个调用，这个时候this就是指向window的，单如果设置了严格模式的话this就是指向undefined

显示绑定：是指通过call、apply、bind进行一个this指向的修改，这三个方法都是接收一个对象或者函数，将原本的this指向转到指定的对象

new绑定的this指向其构造函数 ❌  指向新创建的对象身上

箭头函数没有自己的this绑定，会自内而外寻找最近的对象进行this的绑定❌箭头函数如果在对象中声明的话，this指向window，因为对象是在全局作用域中声明的，如果是在函数中声明的箭头函数，那么this指向函数对象

隐式绑定：如果函数作为对象的一个方法，调用这个函数将会指向最近的对象

## js循环的垃圾回收机制

js的垃圾回收机制是一个“分代收集”的机制，将整个数据存储分成新生代区和老生代区，新生代区又分成两半，一半存储满之后才会通过scanvage算法将装满的一半当中的仍然存活的数据复制到另一半，另一半存储满了之后进入老生代区，进入老生代区之后js使用标记清除算法将不再使用到的数据进行一个清除



## 浏览器的事件循环机制

首先将要执行的代码放入执行栈当中，执行栈中的代码执行完毕之后去任务队列当中取出任务，

先取微任务，如promise.then catch  finally, process.next,如果没有微任务之后取宏任务队列当中的任务，如果宏任务执行过程当中有微任务产生，先执行微任务，然后再执行宏任务



## 手写实现promise.all

```
function myAll(promises) {
  // 返回一个新promise
  return new Promise((resolve, reject) => {
    if (!Array.isArray(promises)) {
      return reject(new Error('这个不是数组，无法处理'));
    }

    let result = [];
    let completedCount = 0; // 已经处理的promise个数

    // 空数组：直接resolve空数组（符合Promise.all标准）
    if (promises.length === 0) {
      return resolve(result); // ✅ 修复：resolve而非return
    }

    // 遍历每一个promise
    promises.forEach((promise, index) => {
      // 兼容普通值（非Promise）
      Promise.resolve(promise)
        .then((value) => {
          completedCount++;
          result[index] = value; // 保证结果顺序和原数组一致

          // 所有Promise都完成时，resolve结果数组
          if (completedCount === promises.length) {
            resolve(result);
          }
        })
        .catch((reason) => {
          // 只要有一个失败，立即reject（符合Promise.all的“快速失败”特性）
          reject(reason);
        });
    });
  });
}
```

## 手写模拟new

我们使用new操作符，语言进行了这些操作：创建一个空对象 -> 让空对象的_proto_指向构造函数的prototype -> 调用构造函数指向该对象 -> 如果构造函数返回一个对象，则返回该对象；否则返回第一步创建的空对象。

```
function mynew(Func, ...args) {
 // 1.创建⼀个新对象
 const obj = {}
 // 2.新对象原型指向构造函数原型对象
 obj.__proto__ = Func.prototype
 // 3.将构建函数的this指向新对象
 let result = Func.apply(obj, args)
 // 4.根据返回值判断
 return result instanceof Object ? result : obj
}
```

function myNew(Func,...args){

	return result instanceof Object ? result : obj
}

```
/**
 * 数组转树形结构
 * @param {Array} nums 原始数组，每项需包含 id、parentId 字段
 * @returns {Object|null} 树形结构的根节点（若多个根节点返回数组，单根节点返回对象）
 */
function arrayToTree(nums) {
  if (!Array.isArray(nums) || nums.length === 0) return null;

  const map = new Map(); // 存储 id -> 节点的映射（方便快速查找父节点）
  const roots = []; // 存储所有根节点（兼容多根节点场景）

  // 第一步：遍历数组，初始化所有节点并存入map
  for (const item of nums) {
    // 跳过无效项（无id的项）
    if (item.id === undefined || item.id === null) continue;
    // 解构节点，初始化children数组
    const node = { ...item, children: [] };
    map.set(node.id, node);
  }

  // 第二步：遍历map，建立父子关系
  for (const node of map.values()) {
    const parentId = node.parentId;
    // 根节点：parentId为null/0/undefined，且自身id不等于parentId（避免自环）
    if (parentId === null || parentId === 0 || parentId === undefined) {
      roots.push(node);
      continue;
    }

    // 找到父节点并挂载子节点
    const parentNode = map.get(parentId);
    if (parentNode) {
      parentNode.children.push(node);
    }
  }

  // 兼容：单根节点返回对象，多根节点返回数组
  return roots
}
```



## http版本区别

HTTP是客户端与服务端之间的通信协议，主要用于数据交互，目前分为HTTP/1.0、HTTP/1.1、HTTP/2、HTTP/3四个版本：

1. HTTP/1.0是短连接：每次请求都需要新建TCP连接（三次握手），请求完成后关闭连接（四次挥手），高并发场景下TCP握手/挥手的资源消耗大、延迟高；

2. HTTP/1.1支持长连接：多个请求可复用同一个TCP连接，但同一连接内的请求是“串行传输”，会导致HTTP应用层的队头阻塞（前一个请求超时，后续请求被阻塞）；

   HTTP/2支持多路复用：将每个请求封装为独立的“流”，在同一TCP连接内并行传输，同时通过HPACK算法压缩请求头，彻底解决了HTTP应用层的队头阻塞，但TCP传输层的队头阻塞依然存在（数据包丢失会导致整个连接的流被阻塞）； 

   HTTP/3基于QUIC协议（QUIC是基于UDP实现的可靠传输协议）：通过“流级独立传输”，彻底解决了TCP层的队头阻塞，同时支持0-RTT连接建立、连接迁移等特性，适配弱网/移动场景。

Vue-Router的核心是实现URL和视图的双向闭环映射，我的设计思路如下： 核心依托`routes`配置表建立路径与组件的映射关系，一方面，URL变化时（浏览器前进/后退、手动改地址栏），Hash模式监听`hashchange`事件提取`hash`路径，History模式监听`popstate`事件提取`pathname`，匹配配置表后更新响应式路由状态，`<router-view>`通过`h`函数渲染对应组件，完成“URL变→视图变”；另一方面，视图触发跳转时（点击`<router-link>`），Hash模式直接修改`window.location.hash`，History模式调用`history.replaceState`（path）主动更新URL，浏览器会在栈的「顶部」新增一条对应 `path` 的历史记录，栈指针指向这条新记录，然后使用`window.history.go()/back()`实现历史记录管理，同时同步更新路由状态，匹配新组件渲染，实现“视图变→URL变”。整个过程通过响应式状态衔接，两种模式分别借助浏览器原生事件/API完成URL管理，最终形成视图和URL的闭环关系。

4、如果断网重连之后如何保持消息的连贯性

当由于网络异常、服务器关闭等断开了SSE连接，浏览器原生SSE默认重试并且携带上last-event-id断点续传，服务端可以通过retry字段规定重连时间间隔，但是为了排除浏览器重连异常导致的无限重连的资源浪费情况，我们封装了重连hook来保证内容的连续性，如果存在链接先调用`es.close()`彻底终止旧链接，释放资源，然后获取最后一个事件的ID，创建新实例url上携带上一次的事件ID实现，同时呢也做了边界控制的优化，通过es的readyState判断链接状态，通过重试次数限制避免无线重连

```
let es;
let retryCount = 0;
const MAX_RETRY = 5; // 最大重试次数

function connectSSE() {
  if (es) es.close();

  const lastId = es ? es.lastEventId : '';
  es = new EventSource(`/sse?lastEventId=${lastId}`);

  es.onmessage = (e) => {
    retryCount = 0; // 收到消息，重置重试次数
    console.log('消息:', e.data);
  };

  es.onerror = () => {
    if (es.readyState === EventSource.CLOSED) {
      retryCount++;
      if (retryCount > MAX_RETRY) {
        console.log('达到最大重试次数，停止重连');
        es.close();
        return;
      }
      // 指数退避重连（1s→2s→4s...），更友好
      const delay = Math.min(1000 * Math.pow(2, retryCount), 10000);
      console.log(`第${retryCount}次重连，延迟${delay}ms`);
      setTimeout(connectSSE, delay);
    }
  };
}

connectSSE();
```

2、虚拟列表的实现

虚拟列表的原理是只渲染可视窗口内的dom节点，分为定高和不定高，关于定高列表的设计就是通过监听scroll事件拿到可视块的scrollTop，然后计算出当前列表中的dom节点的索引，第一个节点的索引是scrollTop/行高 - 缓冲区的数量，这个缓冲区是防止白屏的一种优化手段，同时最后一个节点的索引是 start + 可视高度/行高 + 缓冲区，然后使用transform3d将偏移量也就是第一项的start，然后生成html进行渲染。关于不定高的渲染也需要维护一个元位置数据表，包括每一项的top,buttom,height,index,先固定每一项的高度进行占位，后续动态更新每一项的高度。主要实现先查找到数据表中第一个bottom>scrollTop的作为第一项，将可视区域内的数据slice截取出来，进行渲染，后续使用resizeOberser动态获取到dom节点的高度进行更新，后续的每一项都需要加上更新的差值。

 4、vue2和vue3响应式原理区别

vue2是基于object.defineProperty的，vue3是基于proxy的，vue2的原理是对每一个属性进行监听，这样会导致无法监听到新增属性，而vue3使用的是整个对象的代理，可以监听到新增和删除属性。整个响应式的原理是通过get获取到属性值，通过set修改属性值，这只是在数据层面，然后需要使用track进行依赖收集，trigger实现依赖更新。

新增通过set拦截器，删除通过es6的一个新增内置对象Reflect实现。当然各自有各自的优点，vue3的proxy对象不支持ie，可以使用vue2或者是应用polyfill降级处理
