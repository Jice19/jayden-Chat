# RAG 优化任务拆解（评审版）

## 0. 任务目标
- 完成摘要索引驱动的 RAG 改造：
  - 文档递归切分
  - 摘要向量检索
  - 原文关联存储
  - 多路召回
  - Qwen 重排
  - Top5 上下文生成

## 1. 任务分阶段

### Phase 1：数据结构与基础能力（P0）
- [ ] 1.1 设计并落地 `rag_chunk` 与 `rag_summary_index` 数据模型
- [ ] 1.2 明确 `document_uuid/chunk_uuid/summary_uuid` 生成与关联规则
- [ ] 1.3 为检索链路补充必要索引（`user_id`、`document_uuid`、向量索引）
- [ ] 1.4 增加类型定义（TS interfaces/types），避免隐式 `any`

交付物：
- Prisma schema / SQL migration
- 类型定义文件与映射函数

验收标准：
- 可查询到任意摘要对应的原文切片与所属文档

### Phase 2：文档入库链路改造（P0）
- [ ] 2.1 接入 LangChain `RecursiveCharacterTextSplitter`
- [ ] 2.2 配置切分参数（`chunkSize`、`chunkOverlap`）并可配置化
- [ ] 2.3 针对每个 chunk 生成摘要与关键词
- [ ] 2.4 摘要向量写入向量库，原文写入文档存储（PG 优先，Redis 可选）
- [ ] 2.5 入库链路增加幂等与失败重试策略

交付物：
- 入库服务模块（split/summarize/embed/store）
- 入库状态日志

验收标准：
- 上传单篇文档后，可看到 chunk、summary、关联记录完整落库

### Phase 3：问答检索链路改造（P0）
- [ ] 3.1 对用户问题做清洗与安全过滤
- [ ] 3.2 使用 AI 生成 3~5 条相似问题（多路召回 Query Expansion）
- [ ] 3.3 并行检索摘要向量，合并去重后固定 Top-K=20
- [ ] 3.4 通过 `summary_uuid -> chunk_uuid` 映射回原文切片
- [ ] 3.5 使用摘要关键词做原文匹配过滤

交付物：
- `ask` 检索前半链路服务
- 检索中间结果对象（queries、top20、matched chunks）

验收标准：
- 请求返回中可观测到多路 query 与 Top20 候选

### Phase 4：重排与生成链路（P0）
- [ ] 4.1 接入 Qwen 重排模型（Reranker）
- [ ] 4.2 将候选文档按相关性重排并截断 Top5
- [ ] 4.3 Prompt 模板改造为只注入 Top5 文档上下文
- [ ] 4.4 输出答案时附上引用（document/chunk 信息）

交付物：
- 重排服务模块
- 新版 Prompt 模板

验收标准：
- 最终上下文严格为重排后 Top5，回答可追溯到引用

### Phase 5：接口、监控与回归（P1）
- [ ] 5.1 扩展 `POST /api/rag/ask` 入参与响应结构
- [ ] 5.2 增加核心指标埋点（召回耗时、重排耗时、最终总耗时）
- [ ] 5.3 增加错误码与降级策略（重排失败时兜底）
- [ ] 5.4 补充关键测试用例（入库、检索、重排、生成）
- [ ] 5.5 运行 `pnpm lint` 与类型检查，修复问题
- [ ] 5.6 增加 `debug=true` 调试模式，返回结构化“过程可视化”信息
- [ ] 5.7 增加调试模式安全控制（默认关闭、白名单、脱敏）

交付物：
- API 文档更新
- 埋点日志与测试报告

验收标准：
- 回归通过，功能可稳定运行
- `debug=true` 时可看到 query 扩展、Top20 召回、重排分数、Top5 上下文
- 调试响应不暴露模型原始思维链与敏感信息

## 2. 实施优先级
- P0（必须先做）：Phase 1~4
- P1（上线前完善）：Phase 5

## 3. 任务依赖关系
- Phase 1 -> Phase 2 -> Phase 3 -> Phase 4 -> Phase 5
- 若需要并行：
  - 数据建模（Phase 1）与 Prompt 草案设计可并行
  - 监控埋点（Phase 5）可在 Phase 3 后提前介入

## 4. 风险清单
- [ ] 摘要质量低导致误召回
- [ ] 多路召回噪音过高导致重排压力增大
- [ ] 重排模型延迟抖动影响整体响应时延
- [ ] Redis/PG 双存储一致性问题

对应措施：
- [ ] 增加摘要关键词与原文匹配双重过滤
- [ ] 控制 Query Expansion 条数与 TopK 上限
- [ ] 增加重排超时降级策略
- [ ] 首版采用 PG 主存储，Redis 仅做缓存
